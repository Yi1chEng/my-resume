一个现代化、响应式的个人简历展示网站。采用最新的前端技术栈，展示专业的技能、项目经历和学术成就。

## ✨ 功能特性

- 🎨 **现代化设计** - 使用 Chakra UI 构建优雅的用户界面
- 🌓 **深浅主题切换** - 支持亮色/暗色模式，改善用户体验
- 📱 **响应式布局** - 完美适配桌面、平板和手机屏幕
- ⚡ **高性能构建** - 基于 Vite，极速开发体验和生产构建
- 📝 **易于维护** - 数据配置分离，修改 `src/data.ts` 即可更新所有内容
- 🔍 **SEO 友好** - 优化的文档结构和元数据
- 📊 **交互式组件** - 丰富的 UI 组件，包括幻灯片、进度条、时间线等

## 🚀 快速开始

### 前提条件
- Node.js >= 16.0
- npm 或 yarn

### 安装和运行

1. **克隆项目**
   ```bash
   git clone https://github.com/Yi1chEng/my-resume.git
   cd my-resume
   ```

2. **安装依赖**
   ```bash
   npm install
   ```

3. **配置你的简历数据** ⭐
   
   这是最重要的一步！编辑 `src/data.ts` 文件，将所有内容替换为你的信息：
   
   ```typescript
   // src/data.ts - 修改以下内容为你的信息
   export const profile = {
     name: "你的名字",
     nameEn: "Your Name",
     title: "你的职位/专业",
     // ... 继续修改其他字段
   }
   ```
   
   **关键字段说明：**
   - `profile` - 基本信息（姓名、职位、标签、PDF 链接等）
   - `social` - 社交媒体链接（GitHub、LinkedIn、邮箱等）
   - `about` - 个人介绍和数据统计
   - `education` - 教育背景和证书
   - `codeCard` - 代码卡片展示

4. **添加简历 PDF 文件**
   
   将你的简历 PDF 文件放在 `public/` 目录下，然后在 `src/data.ts` 中配置路径：
   
   ```typescript
   export const profile = {
     // ...
     resumePdf: "/my-resume/resume.pdf",  // 修改为你的 PDF 路径
     // ...
   }
   ```
   
   > 💡 **提示：** 
   > - PDF 文件建议放在 `public/` 文件夹中
   > - 确保 `resumePdf` 的路径与实际文件位置一致
   > - 如果不需要 PDF 下载，可以设置为 `""`

5. **启动开发服务器**
   ```bash
   npm run dev
   ```
   访问 `http://localhost:5173` 查看你的简历网站

6. **构建生产版本**
   ```bash
   npm run build
   ```

7. **预览生产构建**
   ```bash
   npm run preview
   ```

## 🎯 自定义你的简历

### 修改简历内容

所有的简历数据都存储在 `src/data.ts` 文件中。只需编辑这个文件即可自动更新网站上的所有内容：

```typescript
// src/data.ts
export const profile = {
  name: "你的名字",
  title: "你的职位",
  tags: ["技能1", "技能2", "技能3"],
  // ...
}

export const education = {
  timeline: [
    {
      period: "年份范围",
      title: "学位 · 专业",
      institution: "学校名称",
      description: "描述",
      tags: ["标签1", "标签2"],
    },
    // ...
  ],
  // ...
}
```

### 修改主题

编辑 `src/theme.ts` 文件自定义颜色、字体等样式。

### 添加社交链接

在 `src/data.ts` 中的 `social` 对象中添加或修改社交媒体链接。

## 🔧 开发指南

### 代码风格

- 使用 TypeScript 编写代码，避免 `any` 类型
- 遵循 ESLint 规则检查
- 使用有意义的变量和函数命名

### 添加新组件

1. 在 `src/components/` 目录下创建新组件文件
2. 使用 TypeScript 进行类型定义
3. 导入 Chakra UI 组件构建 UI
4. 在需要的地方导入和使用

### 修改样式

使用 Chakra UI 的 `sx` 属性或 Emotion 的 `css` 函数进行样式定义，保持代码的可维护性。
